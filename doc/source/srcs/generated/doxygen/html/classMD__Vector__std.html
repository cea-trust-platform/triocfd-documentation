Detailed Description

.. Il supporte la notion d'items "communs" (sommets du maillage partages entre plusieurs processeurs) et d'items "distants" et "virtuels" (sommets, elements dans l'epaisseur de joint).

See also
MD_Vector_composite (descripteur compose d'une reunion de plusieurs descripteurs, pour le P1Bulle par exemple)

Constructeur.

Si nb_items_reels >= 0, items_to_compute contiendra un bloc avec les items reels, sinon, items_to_compute prendra tous les items (jusqu'a nb_items_tot).

Parameters
(nb_items_tot)valeur que prendra nb_items_tot_ (doit etre >= 0)
(nb_items_reels)valeur que prendra nb_items_reels_ (doit etre >= -1, -1 signifie qu'il n'y a pas d'items "reels" groupes au debut du tableau
(pe_voisins)liste des processeurs voisins, classes dans l'ordre croissant (les tableaux ArrsOfInt doivent avoir la meme taille). Cette liste n'est pas reprise integralement: on retire les processeurs pour lesquels toutes les trois listes d'items sont vides.
(items_to_send)pour chaque pe_voisin, liste des items a envoyer (communs et virtuels). count_items_to_send_to_items_ est calcule par une communication a partir de items_to_recv.
(items_to_recv)liste d'items individuels a recevoir des differents procs (souvent des items communs mais ce n'est pas obligatoire, ex front-tracking)
(blocs_to_recv)liste de blocs d'items a recevoir (voir MD_Vector_std::blocs_to_recv_) (souvent les items virtuels...)
